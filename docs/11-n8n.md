# 11. (オプション) n8n で家庭用自動化ハブを立てる

前: [10. 各クライアントからの接続確認](10-verification.md)

## 位置付け

§10 までで家庭用ファイルサーバの基本構成は完成している。この章は**必須ではない追加機能**で、外部サービス (Slack, メール, カレンダー, Webhook, RSS 等) と Nextcloud を連携させたくなった時に、その「配管」を担うサービスとして [n8n](https://n8n.io/) を同じ Docker ホストに立てる手順をまとめる。

イメージとしては**セルフホスト版 Zapier / Make**。ノードエディタでワークフローを GUI で組み、トリガー (Slack 投稿, Webhook 受信, cron, etc.) と アクション (Nextcloud に保存, メール送信, HTTP 呼び出し, etc.) を繋ぐ。

## 目的

- n8n を Docker で立て、`https://n8n.home.lan` (NPM 経由 / LAN・VPN から) で操作できるようにする
- 初回ワークフローとして「Slack チャンネルに投稿された添付ファイルを自動で Nextcloud の指定フォルダに保存」を動作させる
- 以後のワークフロー追加の土台とする

## 前提

- [10. 確認](10-verification.md) までが一通り通っている (AGH / NPM / Nextcloud / WireGuard が動いている)
- AdGuard Home ([§6](06-adguard-home.md)) に `n8n.home.lan` → `<OMV_IP>` を追加できる (既にワイルドカードエントリを入れてあれば不要)
- Slack ワークスペースの管理権限 (Slack App を作るため)

## 手順

### 11-1. LAN 内 DNS に `n8n.home.lan` を追加

[§6-5 の DNS rewrites](06-adguard-home.md) に以下を追加 (既にワイルドカード `*.home.lan` を入れていれば自動的に解決されるのでスキップ可):

- `n8n.home.lan` → `<OMV_IP>`

### 11-2. compose ファイルと .env の作成

`/srv/compose/n8n/.env`:

```bash
sudo mkdir -p /srv/compose/n8n
sudo nano /srv/compose/n8n/.env
```

```dotenv
SSD_ROOT=/srv
TZ=Asia/Tokyo

N8N_HOST=n8n.home.lan
N8N_PROTOCOL=https
N8N_PORT=5678
WEBHOOK_URL=https://n8n.home.lan/

# 認証 (初回アクセス時に自分で管理者アカウントを作る方式を使う。
#  以下は「保存されたクレデンシャルを暗号化する鍵」。1回決めたら変えないこと)
N8N_ENCRYPTION_KEY=<openssl rand -hex 32 の出力>
```

`N8N_ENCRYPTION_KEY` は以下で生成:

```bash
openssl rand -hex 32
```

保存後にパーミッション:

```bash
sudo chmod 600 /srv/compose/n8n/.env
```

`/srv/compose/n8n/docker-compose.yml`:

```bash
sudo nano /srv/compose/n8n/docker-compose.yml
```

```yaml
services:
  n8n:
    image: n8nio/n8n:latest
    container_name: n8n
    restart: unless-stopped
    environment:
      TZ: ${TZ}
      N8N_HOST: ${N8N_HOST}
      N8N_PROTOCOL: ${N8N_PROTOCOL}
      N8N_PORT: ${N8N_PORT}
      WEBHOOK_URL: ${WEBHOOK_URL}
      N8N_ENCRYPTION_KEY: ${N8N_ENCRYPTION_KEY}
      # NPM の背後にいるので proxy hop を 1 信頼
      N8N_PROXY_HOPS: 1
    volumes:
      - ${SSD_ROOT}/appdata/n8n:/home/node/.n8n
    networks:
      - proxy-net

networks:
  proxy-net:
    external: true
```

n8n のデータ (設定 / クレデンシャル / ワークフロー定義) は小さく変更が多いので SSD 側に置く。

### 11-3. 起動

```bash
sudo -i
cd /srv/compose/n8n
docker compose up -d
docker compose logs -f n8n
exit
```

`Editor is now accessible via: http://0.0.0.0:5678/` と出れば起動完了。

### 11-4. NPM でプロキシホストを追加

NPM UI (`http://<OMV_IP>:81/`) → `Hosts → Proxy Hosts → Add Proxy Host`:

| 項目 | 値 |
|---|---|
| Domain Names | `n8n.home.lan` |
| Scheme | `http` |
| Forward Hostname/IP | `n8n` |
| Forward Port | `5678` |
| Block Common Exploits | On |
| Websockets Support | **On** (n8n の UI は WebSocket を使うので必須) |

`SSL` タブ:
- SSL Certificate: [§7-4](07-nginx-proxy-manager.md) の自己署名を流用する場合、CN/SAN に `n8n.home.lan` を含めた新しい証明書を作り直すか、`*.home.lan` ワイルドカードで再発行するのが楽
- Force SSL: On
- HTTP/2 Support: On

Save。

> **TIP**: 家庭内で複数サービスを増やすなら、最初から `*.home.lan` のワイルドカード自己署名を作って NPM に入れておくと、以後サービス追加のたびに証明書を作り直さずに済む。

### 11-5. 初回ログインと管理者作成

ブラウザで `https://n8n.home.lan/` を開く。自己署名の警告を例外許可。

初回は管理者アカウント作成画面が出る:

- Email / First name / Last name / Password を設定

ログイン後にダッシュボードが表示されれば OK。

### 11-6. Nextcloud への書き込み用クレデンシャルを準備

n8n の Nextcloud ノードが Nextcloud に書き込むための認証情報を作る。**管理者アカウントは使わず、専用ユーザーを作る**のが作法的に安全 (事故ってもそのユーザーに閉じる)。

Nextcloud にログイン (自分の普段使いアカウントでよい) し、`Settings → Security → Devices & sessions` で **App password を生成**:

- Name: `n8n` 等
- `Create new app password` → 表示された**パスワードをその場でコピー** (以後再表示されない)

このパスワードを n8n に渡すことで、Nextcloud 本体のログインパスワードを n8n に保存せずに済む。パスワードが漏れた場合も同画面から取り消せる。

### 11-7. ワークフロー例: Slack チャンネル → Nextcloud

#### 11-7-1. Slack App の作成

1. <https://api.slack.com/apps> → **Create New App** → **From scratch**
2. App Name: `nextcloud-archiver` 等、Workspace を選択
3. 左メニュー **OAuth & Permissions** → **Scopes → Bot Token Scopes** に以下を追加:
   - `files:read`
   - `channels:history` (対象チャンネルが public の場合)
   - `groups:history` (private channel の場合)
   - `channels:read`
4. 左メニュー **Event Subscriptions** → **Enable Events: On**
   - **Request URL**: 後で n8n 側から取得した Webhook URL を貼る (次節で戻ってくる)
   - **Subscribe to bot events**: `file_shared` を追加
5. **Install to Workspace** でインストール → Bot Token (`xoxb-...`) を控える
6. Slack の対象チャンネルで `/invite @nextcloud-archiver` して Bot を参加させる

#### 11-7-2. n8n 側のワークフロー作成

n8n ダッシュボード → **Workflows → New**:

1. **Slack Trigger ノード** を追加
   - Credential: 新規作成 → Bot Token に上記 `xoxb-...` を貼る
   - Trigger On: `File Shared`
   - ノードを Save すると Webhook URL が表示される → Slack App の **Event Subscriptions → Request URL** に貼って Verify
2. **Slack (files.info) ノード** を繋ぐ
   - Resource: File, Operation: Get
   - File ID: 前ノードの `{{$json.event.file_id}}`
   - これでファイルのメタ情報 (URL 等) が取れる
3. **HTTP Request ノード**でファイル本体をダウンロード
   - URL: `{{$json.url_private_download}}`
   - Authentication: Header Auth (Header Name: `Authorization`, Value: `Bearer xoxb-...`)
   - Response Format: File
4. **Nextcloud ノード**を繋ぐ
   - Credential: 新規作成 → Web URL `https://nextcloud.home.lan`、User `<自分のユーザー名>`、Password に §11-6 で作った App password
   - Resource: File, Operation: Upload
   - Path: `/slack-archive/{{$now.format("yyyy-MM")}}/{{$node["Slack1"].json.name}}`
   - Binary Data: On (前ノードの出力を使う)
5. ワークフロー右上の **Active** をオン

#### 11-7-3. 動作確認

Slack の対象チャンネルに画像やファイルをドラッグ投稿 → 1 分以内に Nextcloud の `/slack-archive/2026-04/` 以下に同じファイルが現れれば成功。

n8n のダッシュボード左メニュー **Executions** で実行ログが見られる。失敗していれば各ノードの赤マークを辿って原因を確認。

## 他に作れる自動化の例

一度土台ができれば、以下も同じ要領で組める:

- **RSS 新着 → Nextcloud Notes に追記** (読みたい記事の蓄積)
- **Google カレンダー予定 → Nextcloud Talk に通知** (朝の予定告知)
- **特定メールの添付 → Nextcloud の `/メール保管` に振り分け**
- **毎日 02:30 に Nextcloud の空き容量をチェック → 閾値超えで Talk に投稿**
- **Web フォーム (n8n の Webhook Trigger) → Nextcloud Forms 代替**

n8n は 400+ 種のノードが標準搭載されており、「トリガー何か / 加工 / アクション何か」の 3 点セットで考えると発想しやすい。

## 確認

- [ ] `sudo docker ps` で `n8n` コンテナが `Up`
- [ ] `https://n8n.home.lan/` でログインできる
- [ ] Slack → Nextcloud のワークフローが Active で、対象チャンネルへの投稿がサーバ側に保存される
- [ ] `/srv/appdata/n8n/` 以下に SQLite DB とワークフロー定義が生成されている (バックアップ対象に入っていることを [§9](09-backup.md) で確認)

## バックアップへの追加

n8n の設定/クレデンシャル/ワークフローは `/srv/appdata/n8n/` に入る。[§9](09-backup.md) のバックアップ対象が `/srv/appdata/` 全体を拾うなら何もしなくてよいが、個別指定している場合は `n8n/` を対象に追加する。

## トラブルシュート

- **`https://n8n.home.lan/` で 502**: NPM が n8n に到達できていない。`sudo docker network inspect proxy-net` で `n8n` コンテナが参加しているか確認
- **Slack Trigger の Request URL 検証に失敗**: `WEBHOOK_URL` が `.env` で正しく `https://n8n.home.lan/` になっているか、NPM 側で Websockets Support が On か
- **Nextcloud ノードで 401**: App password の値、またはユーザー名の typo。App password は空白なしでコピーする
- **ファイルが壊れて保存される**: HTTP Request ノードで Response Format が File (Binary) になっているかを確認。Text/JSON になっていると中身が base64 化されて壊れる
- **Slack からの event が届くが n8n の Execution が走らない**: ワークフロー右上の **Active** が Off のまま、が定番のミス
